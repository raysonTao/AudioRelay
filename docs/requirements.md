# Audio Relay 需求文档

## 项目概述

将 Android 平板（小米平板5）的系统音频通过 WiFi 局域网实时传输到 Mac（Mac Air M4），由 Mac 连接蓝牙耳机播放。用户可同时听到两台设备的声音。

## 功能需求

### 核心功能
1. **系统音频捕获**：捕获 Android 设备上所有应用的音频输出（音乐、视频、游戏等）
2. **实时传输**：通过 WiFi 局域网将音频数据实时传输到 Mac
3. **音频播放**：Mac 端接收并播放音频，支持蓝牙耳机输出
4. **设备发现**：通过 mDNS/Bonjour 实现零配置设备发现

### 行为要求
- Android 端采集音频时**本地静音**（通过 AudioManager 将媒体音量设为 0）
- 客户端连接时静音，断开时恢复原始音量
- AudioPlaybackCapture 在音量为 0 时仍可正常捕获音频
- Android 端支持后台运行（前台服务 + 常驻通知）
- Mac 端支持后台运行（窗口最小化/隐藏后继续播放）

### 用户界面
- **Android 端**：开始/停止按钮、连接状态、服务器地址、音量指示、静音状态
- **Mac 端**：设备列表、手动 IP 输入、连接状态、缓冲水位、音量指示

## 非功能需求

### 延迟
- 可接受较高延迟（500ms+）
- 优先保证播放顺畅无杂音

### 音质
- 48kHz 采样率、立体声
- Opus 编码 96kbps
- 无杂音、无爆音

### 可靠性
- 断线自动重连（指数退避 1s→30s）
- 网络抖动自适应缓冲（100-300ms）
- 丢包补偿（Opus PLC）

### 兼容性
- Android 10+（API 29+）
- macOS 13+

## 使用场景
1. 在平板上看视频，声音通过 Mac 蓝牙耳机播放
2. 在平板上玩游戏，游戏音效通过 Mac 播放
3. 在平板上听音乐，音乐通过 Mac 蓝牙耳机播放
